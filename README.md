# gsm-scale-firmware

Прошивка для портативных весов на базе ESP32 с передачей данных через GSM-канал (SIM800L).

---

## Аппаратная платформа

### Вычислительное ядро
| Компонент | Функциональное назначение |
|---|---|
| **ESP32-WROOM-32** | SoC с ядрами Xtensa LX6 (до 240 МГц), интегрированными радиоинтерфейсами: 802.11 b/g/n Wi-Fi, Bluetooth 4.2 BR/EDR + BLE. Управляет периферией через GPIO, UART, I2C, SPI. Выполняет бизнес-логику, протоколы связи, обработку данных. |

### Измерительный тракт
| Компонент | Функциональное назначение |
|---|---|
| **Тензодатчик (Load Cell, 4-проводной)** | Преобразует механическую деформацию в изменение сопротивления тензорезисторов моста Уитстона. Выход — дифференциальный аналоговый сигнал в мВ/В. |
| **HX711** | 24-битный дельта-сигма АЦП с программируемым усилителем (PGA 32/64/128). Обеспечивает оцифровку сигнала тензодатчика, подавление сетевых наводок, интерфейс данных через 2-wire (clock/data). |

### Канал передачи данных
| Компонент | Функциональное назначение |
|---|---|
| **SIM800L** | GSM/GPRS-модуль (quad-band: 850/900/1800/1900 МГц). Реализует стек TCP/IP, HTTP/HTTPS, SMS, USSD. Управление через AT-команды по UART (9600–115200 бод). Питание: 3.7–4.2 В, пиковый ток до 2 А. |

### Система питания
| Компонент | Функциональное назначение |
|---|---|
| **4×18650 (Парралельно)** | Батарея 14.8 В (номинал), ~2.5–3.5 А·ч в зависимости от ячеек. Обеспечивает энергопитание всех подсистем. |
| **BMS 4S** | Плата защиты: балансировка ячеек, отсечка по переразряду/перезаряду, ограничение тока КЗ. |
| **DC-DC Buck (MP2307/MP1584)** | Понижающий преобразователь 14.8 В → 5 В/3 А для питания ESP32 и периферии. |
| **DC-DC Buck (MP1584/LM2596)** | Отдельный преобразователь 14.8 В → 4.0 В/2 А для питания SIM800L (требует низкого ESR, конденсатор ≥1000 мкФ на выходе). |

## Распиновка (ESP32 DevKit v1)

### HX711
| Пин | GPIO | Режим | Примечание |
|---|---|---|---|
| VCC | 3.3V | — | Отдельная линия, не от 5V |
| GND | GND | — | Общая земля |
| DT | GPIO 4 | INPUT | Данные, подтяжка не требуется |
| SCK | GPIO 5 | OUTPUT | Тактирование, частота ≤100 кГц |

### SIM800L
| Пин | GPIO | Режим | Примечание |
|---|---|---|---|
| VCC | 4.0V (DC-DC) | — | Ток до 2 А, конденсатор 1000 мкФ танталовый |
| GND | GND | — | Общая земля, сечение проводника ≥0.5 мм² |
| TX | GPIO 16 (RX2) | UART_RX | Уровень 2.8 В, совместим с 3.3 В логикой |
| RX | GPIO 17 (TX2) | UART_TX | Требуется делитель 3.3 В→2.8 В (резисторы 10 кОм/20 кОм) |
| PWRKEY | GPIO 25 | OUTPUT_OD | Активный низкий, импульс ≥1.2 с для включения |
| RST | GPIO 26 | OUTPUT_OD (опц.) | Сброс модуля, активный низкий |

### SSD1306 (I2C)
| Пин | GPIO | Режим | Примечание |
|---|---|---|---|
| VCC | 3.3V | — | |
| GND | GND | — | |
| SDA | GPIO 21 | I2C_DATA | Подтяжка 4.7 кОм к 3.3 В (если отсутствует на модуле) |
| SCL | GPIO 22 | I2C_CLK | Подтяжка 4.7 кОм к 3.3 В |

### Питание (4×18650)
```
[18650×4] → [BMS 4S] → 
                     ├─[DC-DC 5V/3A]→ ESP32, HX711, SSD1306
                     └─[DC-DC 4V/2A]→ SIM800L
```

|---|---|---|

## Протокол обмена (JSON)

```json
{
  "device_id": "scale_01",
  "weight_kg": 1.250,
  "battery_mv": 14200,
  "rssi": -73,
  "ts": 1710345678
}
```

| Поле | Тип | Описание |
|---|---|---|
| `device_id` | string | Уникальный идентификатор устройства (задаётся в прошивке) |
| `weight_kg` | float | Масса в килограммах, точность ±0.005 кг после калибровки |
| `battery_mv` | uint16 | Напряжение батареи в милливольтах (измеряется через делитель на ADC1_CH0) |
| `rssi` | int8 | Уровень сигнала GSM в dBm (диапазон: -113…-51) |
| `ts` | uint32 | Unix-timestamp в секундах (источник: `millis()` или NTP при наличии Wi-Fi) |

## Калибровка тензодатчика

1. Установить платформу без нагрузки.
2. Выполнить `scale.tare(N)` — усреднение нулевой точки.
3. Разместить эталонный груз известной массы `M_ref`.
4. Считать сырое значение АЦП: `raw = scale.read_raw()`.
5. Рассчитать коэффициент: `CAL = M_ref / (raw - zero_offset)`.
6. Записать `CAL` в NVS (Non-Volatile Storage) или `#define`.

```cpp
// Пример расчёта
#define KNOWN_MASS_KG 2.000f
#define TARE_SAMPLES 32

void calibrate() {
    float zero = scale.read_average(TARE_SAMPLES);
    Serial.print("Zero offset: "); Serial.println(zero);
    
    Serial.println("Place known mass and press any key...");
    while (!Serial.available());
    
    float raw = scale.read_average(64);
    float factor = KNOWN_MASS_KG / (raw - zero);
    
    Serial.print("Calibration factor: "); Serial.println(factor, 6);
    // Сохранить в NVS или записать в defines
}
```

## Отладка и логирование

```bash
# Сборка с отладочными символами
pio run -e esp32dev -t upload

# Мониторинг последовательного порта
pio device monitor -b 115200

# Фильтрация логов по модулю
pio device monitor --filter esp32_exception_decoder

# Прямой доступ к консоли модуля (пасsthrough)
# В network.cpp: #define SIM800_DEBUG_PASSTHROUGH
# Все байты между ESP32 и SIM800L дублируются в USB-Serial
```

---

> Примечание: Модуль SIM800L не поддерживает 3G/4G. При отсутствии покрытия 2G устройство переходит в режим ожидания с периодическим сканированием сети (интервал настраивается). Для регионов с отключением 2G рекомендуется замена на SIM7000/SIM7600 (LTE-M/NB-IoT) с соответствующей адаптацией сетевого модуля.
